# 로아 오늘의 운세 (비공식 팬메이드)

로스트아크 유저용 **재미용 "오늘의 운세"** 페이지입니다. 닉네임을 넣으면 그날의
재련운·품질운·카드운·골드운·파티원운·직업운을 보여줍니다.

> ⚠️ 비공식 팬메이드입니다. **스마일게이트 / 로스트아크와 아무 관련이 없습니다.**
> 운세는 100% 재미용이며 어떤 예측도 아닙니다. (바넘 효과 기반)

## 구조

- `index.html` — 순수 정적 프론트(빌드 도구 없음). 운세 점수·문구 계산은 전부 여기서.
- `api/fortune.js` — Vercel 서버리스 함수. **로스트아크 공식 API 프록시**.
  API 키는 환경변수에 숨기고, 브라우저는 이 함수만 호출합니다.
- 닉네임이 노출 안 됨 → 키는 `LOSTARK_API_KEY` 환경변수에만 둡니다.

### 운세 로직 요약
- `닉네임 + 날짜(KST) + 카테고리`를 해시 시드로 한 결정적 RNG(xmur3 + mulberry32)
  → **같은 닉네임은 그날 하루 몇 번을 봐도 같은 결과**, 자정(KST)에 갱신.
- 클래스명은 시드에 넣지 않음 → API 조회가 실패해도 그날 점수는 안 흔들림.
  클래스는 **직업운 문구 치환에만** 사용.
- 점수(0~100)는 중앙(영웅~전설)이 흔하고 노말/에스더가 드물게 **중앙 편향**.
- 등급은 로아 장비 등급 **노말 · 희귀 · 영웅 · 전설 · 유물 · 고대 · 에스더** 7단계.
  총운은 "오늘의 운세는 ○○ 등급입니다" 형태로 표시.
- 등급과 문구 톤을 일치(bad: 노말·희귀 / mid: 영웅·전설 / good: 유물·고대·에스더 풀에서만 추출).
- 조회 실패/키 없음이면 닉네임 기운만으로 동작(fallback).

## 배포 순서

### 1) 로스트아크 API 키 발급
1. https://developer-lostark.game.onstove.com 접속 → 스토브 로그인
2. **API Key 발급** (마이페이지 / API 관리)
3. 발급된 토큰 문자열을 복사해 둡니다. (이 값이 `LOSTARK_API_KEY`)

### 2) Vercel에 배포 (GitHub 연동, 권장)
1. https://vercel.com 로그인 → **Add New… → Project**
2. **Import Git Repository** 에서 이 저장소(`loa-fortune`) 선택
3. Framework Preset: **Other** (빌드 설정 불필요, 그대로 Deploy)
4. 배포 후 자동으로 `https://<프로젝트>.vercel.app` 주소가 생깁니다.

### 3) 환경변수 등록 (중요)
1. Vercel 프로젝트 → **Settings → Environment Variables**
2. 추가:
   - Name: `LOSTARK_API_KEY`
   - Value: 1)에서 발급한 토큰
   - Environment: Production (+ Preview 원하면 같이)
3. 저장 후 **Deployments → 최신 배포 → Redeploy** (환경변수는 재배포해야 적용)

> 환경변수를 등록하지 않아도 페이지는 동작합니다. 다만 캐릭터 클래스/아이템레벨
> 조회가 빠지고 **닉네임 기운만으로** 보는 fallback 모드가 됩니다.

## 보안 / 남용 방어 (경량)
`api/fortune.js`에 외부 의존성 없는 best-effort 방어가 들어 있습니다.
- **GET만 허용** (그 외 405)
- **입력 검증**: 닉네임은 한글/영문/숫자 1~24자만 통과. 그 외는 상류 호출 없이 fallback 처리 → 인젝션/쓰레기 입력 차단
- **rate limit**: IP당 분당 20회 / 인스턴스 전체 분당 200회 초과 시 429 (로아 토큰 한도 보호)
- **캐시**: 같은 닉네임 응답을 5분간 엣지 캐시 → 반복 호출이 상류로 안 감
- 프론트는 외부 유래 문자열(직업명 등)을 `innerHTML`에 넣기 전 escape

> ⚠️ 서버리스는 인스턴스가 분산되면 메모리 기반 rate limit이 완벽하진 않습니다.
> 한 클라이언트가 닉을 바꿔가며 두드리는 naive 남용은 막지만, 정밀한 분산 제한이
> 필요하면 Upstash Redis 등 외부 저장소 연동이 필요합니다.

## 결과 공유
- **결과 복사**: 텍스트로 클립보드 복사 (키 불필요)
- **이미지 저장**: 결과 카드를 PNG로 저장 (html2canvas CDN, 키 불필요)
- **공유**: 모바일 네이티브 공유 시트 (Web Share API). 카카오톡/인스타 등이 시트에 뜸. 가능한 기기에서만 버튼 노출
- **카카오톡** (선택, 키 필요): 아래 설정 시 버튼 노출

### 카카오톡 공유 켜기 (선택)
1. https://developers.kakao.com → 애플리케이션 추가
2. **앱 키 → JavaScript 키** 복사 (공개돼도 되는 클라이언트 키)
3. `index.html` 상단의 `const KAKAO_JS_KEY = "";` 에 키를 넣기
4. Kakao 콘솔 → **앱 설정 → 플랫폼 → Web** 에 배포 도메인 등록
   (예: `https://<프로젝트>.vercel.app`, 커스텀 도메인 쓰면 그것도)
5. (메시지 전송 시) **카카오 로그인 → 활성화** 및 **카카오톡 공유** 사용 설정

> 키를 비워두면 카카오톡 버튼은 자동으로 숨겨지고 나머지 공유는 그대로 동작합니다.

## 방문자 통계 (Vercel Web Analytics)
`index.html`에 Vercel Web Analytics 스크립트가 들어 있습니다. 집계를 켜려면:
1. Vercel 프로젝트 → **Analytics** 탭 → **Enable**
2. 재배포(또는 다음 배포)부터 방문자/페이지뷰가 잡힙니다.

> 콘텐츠 차단기(애드블록)가 있으면 일부 집계가 빠질 수 있습니다. 로컬에서는
> `/_vercel/insights/script.js` 가 404로 떠도 정상(배포 환경에서만 동작).

## 로컬에서 돌려보기 (선택)
정적 파일이라 `index.html`만 열어도 화면은 뜨지만, `/api/fortune` 호출은
Vercel 런타임이 필요합니다. 함께 테스트하려면:

```bash
npm i -g vercel      # 최초 1회
vercel dev           # http://localhost:3000
```

`vercel dev` 사용 시 프로젝트 루트에 `.env.local` 을 만들고
`LOSTARK_API_KEY=발급받은키` 를 넣으세요. (`.gitignore` 에 이미 포함, 커밋 금지)
